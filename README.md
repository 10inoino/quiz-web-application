# クイズ Web アプリ

フジテレビ系クイズ番組「**クイズ！ヘキサゴン**」の初期コーナー「**バッテンチョイス**」のゲーム進行をサポートするための、プレゼンテーション用 Web アプリケーションです。  
問題を PDF 形式で管理し、ブラウザ上でタイマー付きのクイズ進行を行います。

クイズのルールについては [RULES.md](./RULES.md) を参照してください。

| ソフトウェア | バージョン |
|---|---|
| Node.js | 22 以上 |
| npm | Node.js に同梱 |
| Microsoft PowerPoint / Keynote / LibreOffice Impress | PDF 書き出しが可能なもの |

## ローカル環境の立ち上げ

```bash
# 1. 依存パッケージをインストール
npm install

# 2. 開発サーバーを起動
npm run dev
```

起動後、ブラウザで [http://localhost:3000](http://localhost:3000) を開きます。

## パラメーターの設定

`lib/config.ts` を編集することで、アプリの挙動を一括で変更できます。

```ts
// lib/config.ts
export const QUIZ_CONFIG = {
  ANSWER_DURATION_SECONDS: 30,      // 1問あたりの回答時間（秒）
  GRID_COLS: 7,                     // 問題一覧グリッドの列数
  GRID_ROWS: 6,                     // 問題一覧グリッドの行数
  FAST_TIMER_THRESHOLD_SECONDS: 25, // 倍速タイマー音に切り替わる経過秒数
  TIMER_DELAY_SECONDS: 5,           // 問題表示後にタイマーが動き始めるまでの秒数
}
```

### 各パラメーターの詳細

| パラメーター | デフォルト | 説明 |
|---|---|---|
| `ANSWER_DURATION_SECONDS` | `30` | 問題表示中のタイマーが 0 になるまでの秒数 |
| `GRID_COLS` | `7` | 問題一覧画面の列数 |
| `GRID_ROWS` | `6` | 問題一覧画面の行数 |
| `FAST_TIMER_THRESHOLD_SECONDS` | `25` | この秒数を経過するとタイマー音が倍速に切り替わる |
| `TIMER_DELAY_SECONDS` | `5` | 問題 PDF が表示されてからタイマーが動き始めるまでの猶予時間 |

### グリッドと問題数の関係

- **問題数 ≦ `GRID_COLS × GRID_ROWS`** → 正常動作。グリッドに空きセルが生じても問題なし
- **問題数 > `GRID_COLS × GRID_ROWS`** → 起動時にエラーが表示されます。`GRID_COLS` / `GRID_ROWS` を増やして対応してください

> **設定変更後は開発サーバーを再起動してください。**

## アプリの使い方

### 画面の流れ

```
問題一覧（グリッド）
  → 問題番号表示（Q◯）
    → 問題 PDF 表示（タイマー計測中）
      → "Answer" 表示
        → 答え PDF 表示
          → 問題一覧（グリッド）に戻る
```

### 問題一覧画面

- 問題番号のボタンが並んだグリッドが表示されます
- ボタンをクリックすると、その問題の進行が始まります
- 使用済みの問題はグレーアウトして選択不可になります
- 画面右上のボタン、またはキーボードの `F` キーで全画面表示を切り替えられます

### 問題進行中の操作

- **画面のどこかをクリック** → 次のフェーズへ進みます
- タイマーは問題 PDF 表示後に自動でスタートします（`TIMER_DELAY_SECONDS` 秒の猶予あり）
- タイマーが 0 になると自動でタイマー終了音が鳴ります（クリックで次へ進む必要はありません）

### リセット

問題一覧画面の「リセット」ボタンをクリックすると、使用済みの記録が消去されて全問題が選択可能に戻ります。  
使用済み情報はブラウザの LocalStorage に保存されているため、ページをリロードしても引き継がれます。

## 問題の準備と反映

### ファイル構成のルール

`public/questions/` 配下に問題番号ごとのディレクトリを作り、各ディレクトリに以下の 2 ファイルを置きます。

```
public/
└── questions/
    ├── 1/
    │   ├── question.pdf   ← 問題スライド
    │   └── answer.pdf     ← 答えスライド
    ├── 2/
    │   ├── question.pdf
    │   └── answer.pdf
    └── ...
```

この配置は `npm run split-pdf` コマンドで自動生成できます（手動配置も可能）。

> **問題数の数え方**  
> アプリ起動時に `public/questions/` 配下のディレクトリ数を自動的にカウントして問題数とします。  
> 設定ファイル（`lib/config.ts`）に問題数を記載する必要はありません。  
> ただし、問題数が `GRID_COLS × GRID_ROWS` を超えている場合はエラーになります（グリッドに収まらないため）。

### サンプル PowerPoint を使った問題反映手順

プロジェクトルートに `questions_sample.pptx` というサンプルファイルが含まれています。  
以下の手順に従って問題を反映してください。

#### ステップ 1: PowerPoint を PDF に書き出す

PowerPoint を開いて PDF として書き出します。

**Microsoft PowerPoint の場合**
1. ファイル → 名前を付けて保存
2. ファイル形式で「PDF」を選択
3. ファイル名を `questions.pdf` として、プロジェクトルート（`package.json` と同じ場所）に保存

**Keynote の場合**
1. ファイル → 書き出す → PDF
2. ファイル名を `questions.pdf` としてプロジェクトルートに保存

> **PowerPoint のスライド構成のルール**  
> 奇数番スライド（1, 3, 5...）が「問題」、偶数番スライド（2, 4, 6...）が「答え」となるように作成してください。  
> スライドの総枚数は必ず偶数になるようにしてください。

#### ステップ 2: 問題を展開する

```bash
npm run split-pdf
```

実行すると `public/questions/` の既存ディレクトリが削除され、新しい問題が展開されます。

```
入力 PDF: .../questions.pdf
総ページ数: 98
問題数: 49

public/questions/ を初期化中...
  既存ディレクトリを削除しました。

各問題ディレクトリへの振り分けを開始...
  Q1: ページ 1 → question.pdf, ページ 2 → answer.pdf
  Q2: ページ 3 → question.pdf, ページ 4 → answer.pdf
  ...

完了: 49 問を public/questions/ に展開しました。
```

#### ステップ 3: `lib/config.ts` でグリッドを調整する

問題数に合わせて `GRID_COLS` と `GRID_ROWS` を設定します。  
`GRID_COLS × GRID_ROWS ≧ 問題数` になるようにしてください。

例: 49 問の場合 → `GRID_COLS: 7, GRID_ROWS: 7`（7 × 7 = 49）

#### ステップ 4: 開発サーバーを再起動する

```bash
npm run dev
```

ブラウザで確認すると、グリッドに新しい問題が表示されます。
